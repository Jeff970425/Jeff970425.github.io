Regex Golf

Jul 05 2020

容易观察本题需要匹配的是字典序字符串。

我的想法是“任一字符在该字符串内不会在相隔一个及以上的字符后再重复出现”。

这个想法是非完备的，因为在匹配以下情况时无法得到正确结果：

1、字符串内无重复字符，但还是非字典序，如 ‘jestword’。

2、字符串内同一字符连续出现三次及以上，如 ‘abbbc’。

网上大部分正确答案都是cheat的，比如$\wedge.\left\{5\right\}[\wedge e]?\$$，从长度和末尾非e（屏蔽oriole）入手。

实际上，有更直观简单且完备的解法，来自

http://m.udpwork.com/item/11365.html

只需要$\wedge a*b*c*d*e*f*g*h*i*j*k*l*m*n*o*p*q*r*s*t*u*v*w*x*y*z*\$$即可。

github + PicGo + hexo注意事项

关于如何用hexo搭建一个blog的教程网上有很多，这里推荐一个：

小白独立搭建博客–Github Pages和Hexo简明教程

https://www.douban.com/note/545511422/

下面记录我在使用中碰到的几个需要注意的地方：

1、常用命令

新建文章 hexo new “postName”

新建页面 hexo new page “pageName”

启动服务预览 hexo server

生成静态页面 hexo generate

部署 hexo deploy

2、github无法加载图片的解决方法

打开 C:\Windows\System32\drivers\etc\hosts

加上199.232.68.133 raw.githubusercontent.com

cmd ipconfig /flushdns进行刷新

3、购买域名关联到Github Pages

在source目录添加一个新文件CNAME

CNAME不带任何后缀，内容为关联的域名

这样不需要每次hexo d后手动修改GitHub Pages设置

Regex Golf

Jul 02 2020

Regex Golf是个锻炼写正则能力的网站，目前看起来题目数量是无穷的。

今天看到一个很好玩的题目，通过正则匹配质数个x的排列。

接下来解释^(?!(xx+)\1+$)的含义。

^表示字符串开头的位置；

(?!a)表示当前位置后的字符串不能匹配a；

xx+表示大于等于两个连续的x，记长度为n；

?!(xx+)\1+表示（大于等于两个连续的x）连续出现大于等于两次，记次数为m；

$代表字符串结尾的位置。

听起来有点绕，

实际上就是质数的概念。

质数只有1和自己两个因子，

所以(xx+)\1+匹配到的x排列都可以被分解成若干大于1的因子的乘积（n x m），

再确保从字符串边界开始匹配，那么最终表达式的结果自然不是质数。
